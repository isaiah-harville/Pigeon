Security model

Designed to leak as little as possible.

01

No account layer

Pigeon does not require a phone number, email address, social login, or cloud account to create an identity.

02

Keys stay local

Long-term identity keys are generated on-device. Private key material is not uploaded to Pigeon infrastructure.

03

Offline-capable mesh

Nearby devices can exchange messages directly over local radio without a central service in the middle.

04

Relays are blind

Optional relays move opaque ciphertext between mailboxes. They do not receive keys and cannot decrypt message contents.

Built for untrusted networks.

  • End-to-end encryption for message content.
  • Human-verifiable safety numbers for peer identity checks.
  • Local encrypted storage for conversations on device.
  • No ads, tracking SDKs, or analytics pipeline.

Open source

Built in the open.

Pigeon is an open source project. The app, relay, crypto, and mesh packages are developed in public so people can inspect the code, follow the roadmap, file issues, and contribute improvements.
